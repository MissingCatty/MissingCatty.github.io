题目描述

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

你可以按任意顺序返回答案。

想出一个时间复杂度小于 \(O(n^2)\)的算法。

解法一

暴力解法，算法复杂度为\(O(n^2)\)

解法二

在暴力解法中，每次指定其中一个数arr[i]，从而查找剩下来的数组中是否存在target-arr[i]，最耗时的就是这个查找过程，如果线性查找，时间就复杂度为\(O(n)\)。所以，优化的着手点就在查找过程中。

由查找的知识可知，hash表查找某个元素的时间复杂度最好为\(O(1)\)，即查找过程没有冲突，每一个key通过hash函数计算出来的hash值唯一；最坏情况为\(O(n)\)，表中的所有键值对的key计算得到的hash值都恰好相同。所以，最坏情况下该解法时间复杂度才是\(O(n)\)，而在实际使用中，hash表查找最坏情况发生的概率是非常低的。

所以，比较简单暴力的思路就是：在遍历开始前，把数组中每个元素都先存到hash表中，之后查找时直接去表中找。但这种思路在遇到数组中存在两个相同元素的情况时，会出现问题，因为hash表的查找是查找key，而key必须是唯一的。

对上述想法进行改进，遍历到元素arr[i]时，在表中查找target-arr[i]，

  • 如果target-arr[i]在表里，就取出其对应的value（数组下标）
  • 如果不在，就把arr[i]放到表里，如果后面能够遍历到target-arr[i]，就取出arr[i]对应的value

拓展知识

C++中hash表的使用

在C++中，哈希表的实现通常依赖于 unordered_map 容器，它是C++标准库 <unordered_map> 头文件中的一部分。unordered_map 是一个基于哈希表的数据结构，提供了键值对（key-value pair）的存储和查找功能，支持常数时间复杂度的查找、插入和删除操作（在理想情况下）。

插入元素

unordered_map 提供了多种方式来插入元素。常见的插入方式有：

  • 使用 insert() 方法
  • 使用 [] 运算符

查找元素

使用 find() 方法查找元素，返回一个迭代器。如果键存在，返回指向该元素的迭代器，否则返回 end() 迭代器。

删除元素

可以通过 erase() 方法根据键删除元素：

  • 使用键删除单个元素
  • 使用迭代器删除元素

获取元素个数

使用 size() 方法获取 unordered_map 中元素的个数。

判断元素是否存在

使用 count() 方法判断指定的键是否存在，返回一个整数，0表示不存在，1表示存在。
