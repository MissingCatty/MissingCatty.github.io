两个有序数组的中位数

ZYC2024-12-312024-12-31

题目描述

给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的 中位数 。

算法的时间复杂度应该为 O(log (m+n)) 。

示例 2：

解法一

暴力解法，从小到大遍历两个数组，找到第\(\frac{m+n}{2}\)，此方法的时间复杂度为\(O(m+n)\)。

解法二

需要将时间复杂度降为\(O(\log{(m+n)})\)，一般需要用到二分查找。

这道题可以转为：查找两个有序数组中第k小的元素。

如下有两个数组，长度小的数组长度为m=3，长度大的数组长度n=4，现在要找第k=5小元素，其目标就是将两个数组划分为两部分，红线左边的部分有k个数，红线右边有m+n-k个数。第k小元素就是右边部分中的最大的元素。

左右两个部分，都是从两个数组中取若干元素组成的，第一个数组中取i个数，第二个数组中取j个数，满足i+j=k。此外，还需要满足：左边部分中的最大值必须小于等于右边部分中的最小值。由于两个数组都是有序的，所以判断这个条件可以简化为：arr1[i-1] <= arr2[j] && arr1[i] >= arr2[j-1]，即：8必须小于等于10，同时9必须小于等于arr1[3]（这个例子中越界不存在arr[3]，所以边界条件需要额外考虑）。

至此，现在的问题转变为：如何求出上图中的红色分割线？即，在硬性条件“i+j=k”的前提下，如何分配i和j的值（其实也就是求i的值，因为k是定值，j=k-i），使其满足“左边部分中的最大值必须小于等于右边部分中的最小值”这一条件。

i的范围：\([0, \min{(m,k)}]\)

最简单粗暴的办法：将i在上述范围内遍历，找到符合条件的i值。但由于数组有序，所以可以对arr1采用二分查找，来确定i的值。

设立初始时i的范围是left=0, right=min(m,k)，每次循环计算i的值，表达式如下： \[ i = \frac{left+right}{2} \]

  • 如果满足arr1[i-1] <= arr2[j] && arr1[i] >= arr2[j-1]，则得到正确的i和j值
  • 否则，
    • 如果arr1[i-1] > arr2[j]，说明正确的i值在二分后的左半部分[left, i-1]，所以要更新right = i-1
    • 如果arr1[i] < arr2[j-1]，说明正确的i值在二分后的右半部分[i-1, right]，所以要更新left = i+1
